Write a README that gives readers everything they need to understand, install, and use the project.

Standard sections (omit if genuinely not applicable):
- Project name + one-line description
- Badges (CI status, version, licence) — use placeholders if unknown
- Overview / motivation
- Prerequisites
- Installation
- Quick start / usage example
- Configuration reference
- Development setup (how to run tests, lint, build)
- Contributing guidelines
- Licence

Adapt tone and depth to the project type (library vs service vs CLI vs internal tool).
