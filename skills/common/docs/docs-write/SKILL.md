Write clear, accurate, and thorough documentation for the provided code or component.

Guidelines:
- Match the documentation style and format already used in the project (docstrings, JSDoc, godoc, etc.)
- Cover purpose, parameters, return values, exceptions/errors, and usage examples
- Be concise but complete — avoid padding
- If the target is a module or package, include a high-level overview and a quick-start example
